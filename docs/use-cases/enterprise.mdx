## Deployment Architectures

### Cloud-Native Architecture

```
┌─────────────────┐
│  Load Balancer  │
└────────┬────────┘
         │
    ┌────┼────┐
    │    │    │
    ▼    ▼    ▼
  [API][API][API]  ← Auto-scaling
    │    │    │
    └────┼────┘
         │
    ┌────▼────┐
    │  Redis  │  ← Shared cache
    └─────────┘
```

**Benefits:**
- Horizontal scaling
- High availability
- Geographic distribution
- Managed infrastructure

**Setup:**
1. Deploy multiple API instances
2. Configure shared Redis for rate limiting
3. Set up load balancer
4. Enable auto-scaling rules

**Setup:**
1. Deploy Docker containers
2. Configure internal network
3. Set up monitoring
4. Implement backup strategy

### Hybrid Architecture

Mix cloud and on-premise for optimal flexibility.

**Use Cases:**
- Sensitive data → On-premise
- High-volume bulk → Cloud
- Real-time moderation → Edge locations

## Integration Examples

### Enterprise Chat Platform

```python
from guardian_api import GuardianClient
from redis import Redis
import asyncio

class EnterpriseModeration:
    def __init__(self):
        self.client = GuardianClient(
            base_url="https://guardian.internal.corp"
        )
        self.cache = Redis(host='redis.internal.corp')

    async def moderate_message(self, message_id, text):
        # Check cache first
        cached = self.cache.get(f"moderation:{message_id}")
        if cached:
            return cached

        # Moderate
        result = self.client.moderate_text(text)

        # Cache result (1 hour)
        self.cache.setex(
            f"moderation:{message_id}",
            3600,
            json.dumps(result)
        )

        # Log to analytics
        await self.log_moderation(message_id, result)

        return result

    async def bulk_moderate_history(self, messages):
        # Process in batches of 100
        batches = [messages[i:i+100] for i in range(0, len(messages), 100)]

        results = []
        for batch in batches:
            texts = [m.text for m in batch]
            batch_results = self.client.moderate_batch(texts)
            results.extend(batch_results["results"])

        return results
```

### E-Commerce Review System

```javascript
import { GuardianClient } from 'guardian-api-sdk';

class ReviewModerationService {
  constructor() {
    this.client = new GuardianClient({
      baseUrl: 'https://guardian-api.company.com',
      timeout: 30000
    });
  }

  async moderateReview(review) {
    const result = await this.client.moderateText(review.text);

    // Store moderation metadata
    await this.db.reviews.update(review.id, {
      moderationScore: result.ensemble.score,
      moderationSummary: result.ensemble.summary,
      primaryIssue: result.ensemble.primary_issue,
      moderatedAt: new Date()
    });

    // Auto-approve or queue for review
    if (result.ensemble.summary === 'likely_safe') {
      await this.approveReview(review.id);
    } else if (result.ensemble.summary === 'highly_harmful') {
      await this.rejectReview(review.id, result);
    } else {
      await this.queueForReview(review.id, result);
    }

    return result;
  }

  async bulkModerateReviews(productId) {
    const reviews = await this.db.reviews.find({
      productId,
      moderated: false
    });

    const texts = reviews.map(r => r.text);
    const results = await this.client.moderateBatch(texts);

    // Process results
    await Promise.all(
      results.results.map((result, index) =>
        this.processReviewResult(reviews[index], result)
      )
    );
  }
}
```

## Custom Model Training

### Process

<Steps>
  <Step title="Data Collection">
    Provide labeled training data (minimum 10,000 examples)

    Format: CSV with columns `text`, `label`

  <Step title="Model Training">
    We train a custom LASSO classifier on your data

    - Cross-validation
    - Performance benchmarking
  </Step>

  <Step title="Integration">

    - Separate endpoint: `/v1/moderate/custom`

  </Step>
